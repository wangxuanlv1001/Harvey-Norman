隐私与安全要求分析

1. 数据本地存储与临时性

本应用将所有库存数据存储在浏览器的内存中，不发送到任何外部服务器。用户关闭浏览器后数据会丢失，这保证了用户数据的隐私性，不会留下持久记录。适合演示和临时管理场景。

2. 输入验证与防注入

所有用户输入都经过前端验证（Angular 响应式表单），数字字段拒绝非数字输入，文本字段进行了 HTML 转义（使用 Angular 默认的插值保护），防止 XSS 攻击。在更新和删除操作前会进行确认，避免误操作。

3. 唯一标识约束

系统强制 Item ID 和商品名称的唯一性，防止数据重复和逻辑错误。添加或更新时会检查重复并提示用户。

4. 无第三方跟踪

应用未使用任何第三方分析、广告或跟踪脚本，用户行为不会被记录或分享。

5. 移动设备安全建议

在移动设备上使用时，建议用户不要保存敏感商业数据到本地（因为本应用不提供加密存储）。未来版本可增加 IndexedDB 加密存储选项。

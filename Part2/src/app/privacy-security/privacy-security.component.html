Privacy & Security

Your data is handled with care. Learn how we protect your information.

Local Data Storage & Ephemeral Nature

All inventory data is stored in browser memory and never sent to any external server. Data is lost when the browser is closed, ensuring user privacy with no persistent records. Suitable for demo and temporary management scenarios.

Input Validation & Injection Prevention

All user inputs are validated on the front end (Angular reactive forms). Numeric fields reject non-numeric input, and text fields are HTML-escaped using Angular's default interpolation protection, preventing XSS attacks. Confirmation dialogs are used for update and delete operations to avoid mistakes.

Unique Identifier Constraints

The system enforces uniqueness of Item ID and product name to prevent data duplication and logic errors. Checks are performed during add/update and user is notified of duplicates.

No Third-Party Tracking

The application does not use any third-party analytics, advertising, or tracking scripts. User behavior is not recorded or shared.

Mobile Device Security Recommendations

When using on mobile devices, users are advised not to store sensitive business data locally (as this application does not provide encrypted storage). Future versions could add IndexedDB encryption.

✅ This system follows industry best practices for front-end security. No personal data is collected.
